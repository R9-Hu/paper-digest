    OpenSkill: Open-World Self-Evolution for LLM Agents¶

    🕒 Published (v1): 2026-06-04 00:00 UTC · Source: HuggingFace · link

    Why this paper was selected

    OpenSkill: open-world self-evolution without curated skills or verifier signals

    TL;DR¶

    OpenSkill is a three-stage framework for LLM agent self-evolution that requires only a task prompt and public open-world resources—no curated skills, successful trajectories, or verifier signals. It bootstraps a learning loop by retrieving grounded knowledge and verification anchors from documentation/repositories/web, synthesizing transferable skill artifacts, and refining them against self-built virtual test suites. On three benchmarks and two model families, it achieves the best automated pass rate without any target-task supervision during learning.

    Problem¶

    Existing self-evolving agents assume a usable learning loop: curated skills, successful execution traces, task-level rewards, or external verifiers. Real open-world deployments may provide none of these—only a natural-language task prompt. The two coupled gaps are (1) skill construction without access to prior successful trajectories or human-written skills, and (2) verification construction without hidden ground-truth test suites or reward signals. No prior method satisfies both constraints simultaneously.

    Method¶

    OpenSkill formalizes skill evolution as constructing a skill set \(\hat{S}_i = f(I_i, E_i, K)\) from observable input \((I_i, E_i)\) and open-world knowledge \(K\), with the constraint that the hidden ground-truth test suite \(T_i^{GT}\) is never observed during construction.

    Stage 1 — Open-World Knowledge Acquisition. Given instruction \(I_i\) and environment \(E_i\), retrieval function \(D\) queries documentation, repositories, papers, and tutorials to produce knowledge documents \(k_i\) and a structured skill plan \(p_i\). A second retrieval \(D_v\) collects verification knowledge \(k_i^v\)—independently verifiable anchors such as reference values from official docs, dataset row counts, and domain standards—without querying benchmark-identifying strings.

    Stage 2 — Leakage-Free Skill Evolution. The base agent \(\pi_\theta\) generates an initial skill set \(\hat{S}_i^{(0)}\) from \((p_i, k_i)\). A separate verifier LLM session generates a deterministic pytest-based virtual test suite: $\(\tilde{T}_i = g(I_i, E_i, k_i^v)\)$ Skills are iteratively refined using the virtual pass rate: $\(\tilde{r}^{(j)} = \frac{1}{|\tilde{T}_i|} \sum_{k=1}^{K} \tilde{t}_{i,k}\!\left(\pi_\theta(I_i, \hat{S}_i^{(j)}, E_i)\right)\)$ as a proxy for \(T_i^{GT}\). When \(\tilde{r}^{(j)} < 1\), a structured failure diagnostic \(F^{(j)}\) drives refinement: $\(\hat{S}_i^{(j+1)} = \pi_\theta\!\left(\hat{S}_i^{(j)}, F^{(j)} \mid I_i, E_i, p_i, k_i\right)\)$ If the diagnostic identifies a knowledge gap rather than a bug, targeted retrieval \(k_i^{(\text{gap})} = D(F^{(j)}, K)\) is injected. The loop runs for at most \(J=3\) rounds.

    Stage 3 — Zero-Shot Target Evaluation. The frozen artifact \(\hat{S}_i^*\) is deployed to the target agent \(\pi_{\theta'}\) (which may differ from the construction agent), and \(T_i^{GT}\) gates pass/fail only here.

    Key Contributions¶

    • Defines open-world self-evolution: agent must construct both skills and verification signals from scratch using only a task prompt and public resources, with no target-task supervision.
    • Proposes OpenSkill, a three-stage pipeline implementing this setting—knowledge acquisition, leakage-free refinement, zero-shot deployment.
    • Demonstrates cross-model skill transfer: skills built with one model deploy as-is on weaker models (Haiku 4.5, Qwen 3 Coder, DeepSeek V3, Mistral Large 3), improving pass rate by 5.5–14.8 pp over no-skill baseline without any model-specific adaptation.
    • Shows the virtual verifier covers 88.9% of ground-truth test intents (120/135 sampled) and achieves statistically significant alignment with GT outcomes (Fisher's exact OR=2.97, \(p=0.035\)), despite never accessing \(T_i^{GT}\).

    Results¶

    • SkillsBench (primary, 11 domains):
    • Opus 4.6: 43.6% vs. best baseline Skill-Creator 34.7% (+8.9 pp); within 0.9 pp of Human upper bound (44.5%)
    • GPT 5.2: 42.1% vs. best baseline CoT 33.3% (+8.8 pp); within 2.7 pp of Human (44.8%)
    • Best or tied-best in 8/11 domains (Opus) and 7/11 domains (GPT)
    • Largest per-domain gains: Health 69.6%, Software 59.9% (Opus); Energy 80.0%, Cybersecurity 52.5% (GPT, above Human)
    • Manufacturing: 0.0% for all automated methods including OpenSkill
    • SocialMaze: 82.7% (Opus) / 70.7% (GPT); +0.9–2.2 pp over strongest baseline
    • ScienceWorld: 90.0% (Opus) / 85.3% (GPT); best automated in all four columns
    • Ablation (SocialMaze): Open-world query alone +6.1 pp, virtual verifier alone +6.3 pp, both combined +8.2 pp over parametric-only baseline (74.5%)
    • Iteration ablation: Pass rate peaks at 3 refinement iterations (82.7%) and degrades at 5 (79.9%) and 10 (78.0%)

    Limitations¶

    • Manufacturing domain failure: All automated methods score 0.0% on this domain; open-world acquisition does not resolve it.
    • Virtual verifier alignment is imperfect: 60.7% agreement rate with GT outcomes; 56.9% precision / 80.5% recall, leaving meaningful false-positive skill refinement risk.
    • Uncovered test intents: The 11.1% of GT intents not covered cluster in benchmark-specific anti-cheat checks and deep semantic quality properties (taxonomy coherence, lemmatization) requiring domain expertise beyond the task specification.
    • Overfitting to virtual feedback: Excessive refinement iterations degrade performance, implying the virtual proxy diverges from GT at high iteration counts.
    • Context augmentation only: The framework is restricted to context-level skill artifacts; weight modification (fine-tuning, RL) is explicitly excluded, limiting adaptation depth.
    • Cost not fully characterized: Per-task pipeline cost breakdown deferred to appendix; open-world retrieval adds overhead not quantified in the main results.

    Relevance to Agentic AI / LLM Agents¶

    OpenSkill directly addresses one of the core unsolved problems in deployed LLM agents: how to self-improve when no supervision signal is available at runtime. The supervision-free, context-augmentation paradigm—where skills are portable artifacts rather than model weights—offers a practical path to continual agent improvement without retraining, and the cross-model transferability result means a single skill library can serve a heterogeneous agent fleet. The virtual verifier design is particularly significant for the field: it shows that an autonomous agent can construct a meaningful proxy for ground-truth evaluation from public documentation alone, substantially relaxing the assumption that useful self-improvement requires external oracles or human feedback. This connects to a broader line of work on self-evolving agents (Voyager, CRITIC, Reflexion) but is distinguished by operating without any closed-world assumptions during learning.
