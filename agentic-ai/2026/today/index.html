    🆕 Today's Digest — Agentic AI / LLM Agents¶

    New papers/posts collected 2026-06-23.

    19 new today

    📋 In brief¶

    This batch is dominated by two intertwined concerns: the fragility of agent memory and context as trajectories grow long, and the mechanics of training agents to act reliably in the first place. Several papers (SELFCOMPACT, Plans Don't Persist, Kamera, EvoEmbedding, OpenRath) converge on a shared finding that context is not passive storage — plans decay, caches misfire on position shifts, and compaction done blindly breaks task success — making context management a first-class engineering problem rather than a background detail. A parallel thread runs through RL-based training recipes (TMax, G2PO, PhoneBuddy, DataClaw0), each proposing better credit assignment or synthetic environments to push open models toward the performance of closed ones, often at a fraction of the compute. Cutting across both threads is a new emphasis on diagnosing agent failure modes at runtime — premature commitment, parametric over-trust, reward sparsity — with benchmarks (PlanBench-XL, AFTER, Counsel, DailyReport, PhySciBench) that expose how far even frontier systems fall short of "acceptable" on long-horizon, knowledge-intensive, or everyday tasks.

    📝 Today's papers¶

    Tmax: A simple recipe for terminal agents¶

    TMax is an open, reproducible RL training recipe for terminal-using language model agents. It pairs a novel 14,600-task synthetic dataset (TMax-15K) with Divergence Proximal Policy Optimization (DPPO) to train models that outperform all prior open recipes on Terminal-Bench 2.0, with a 9B model reaching 27%—matching closed models like Claude Haiku 4.5.

    📌 Why selected: AI2 (Ivison, Lambert); RL training recipe for terminal agents; directly reproducible

    Group-Graph Policy Optimization for Long-Horizon Agentic Reinforcement Learning¶

    G2PO transforms multi-trajectory RL rollouts into a state-transition graph, enabling group-aggregated state-value estimation and globally-normalized edge-centric advantage signals. This directly addresses reward sparsity and high-variance credit assignment in long-horizon LLM agent tasks. On WebShop and ALFWorld it achieves up to 22.2% absolute success-rate gain over GRPO with negligible additional compute.

    📌 Why selected: Microsoft Research; step-level policy updates extend GRPO to long-horizon agentic RL

    Self-Compacting Language Model Agents¶

    Long agent trajectories accumulate stale content ("context rot") that degrades performance, but existing fixed-interval compaction fires blindly relative to reasoning state. SELFCOMPACT gives the model itself a summarization tool paired with a lightweight rubric specifying when to invoke it, achieving adaptive compaction with no fine-tuning. It matches or exceeds fixed-interval summarization at 30–70% lower token cost across six benchmarks and seven models.

    📌 Why selected: Adaptive self-triggered compaction fixes context overflow in long agent traces

    Plans Don't Persist: Why Context Management Is Load Bearing for LLM Agents¶

    LLM agents do not internalize their plans into persistent hidden state — plans are context-time objects re-read from the window each step, decaying 4.1× in hidden-state cosine distance within a single action-observation cycle. Naive context compression that evicts plan tokens cuts ALFWorld task success by 34.7pp, and plan-protection or probe-gated re-surfacing alone does not recover it. The paper contributes a measurement framework (replay pairing + strict stripping) and a stress test showing context management is load-bearing for agent plan fidelity.

    📌 Why selected: CMU (Datta); diagnoses context management as load-bearing infrastructure for plans

    Kamera: Unified Position-Invariant Multimodal KV Cache for Training-Free Reuse¶

    Multimodal agents repeatedly re-encode the same visual content as context windows slide, frames are reordered, or chunks are evicted and recalled — because existing prefix/radix caches are position-indexed and miss on any offset change. Kamera separates each cached KV chunk into a position-free canonical store plus a small low-rank conditioning patch, enabling exact RoPE re-rotation plus patch replay as a single training-free operator across MLA, GQA, and MHA. This makes sliding-window survival, reordering, and reversible eviction into millisecond cache edits instead of full re-prefills.

    📌 Why selected: Position-invariant KV cache reuse eliminates redundant re-encoding in multimodal agents

    AOHP: An Open-Source OS-Level Agent Harness for Personalized, Efficient and Secure Interaction¶

    AOHP is an open-source OS-level harness built on AOSP that redesigns Android to treat AI agents as first-class OS actors rather than app-layer guests. It introduces three mechanisms—personalized service composition, efficient agent interfaces, and secure information-flow tracking—that together raise task completion by 21.12 pp, cut LLM token consumption by 51.55%, and enforce fine-grained privacy policies across cross-app agent workflows.

    📌 Why selected: Open-source OS-level agent harness; directly actionable for building agent infrastructure

    GIF: Locally Sound Geometric Information Flow Control for LLMs¶

    GIF (Geometric Information Flow) is a principled, quantitative information-flow control (IFC) framework for LLMs that uses the model's Jacobian and local output geometry to upper-bound Shannon mutual information between perturbed input spans and downstream outputs. It avoids the taint-explosion problem of label-based IFC by measuring how much a span influences an output rather than whether it does at all. Combined with lightweight declassifiers, GIF matches or exceeds GPT-5.5 xhigh reasoning at up to 81× lower token cost.

    📌 Why selected: Columbia (Jana); geometric info-flow control prevents prompt injection in agentic pipelines

    Managing Procedural Memory in LLM Agents: Control, Adaptation, and Evaluation¶

    This paper introduces AFTER, a 382-task benchmark for evaluating whether procedural memory (reusable skill instructions distilled from execution traces) in LLM agents generalizes across tasks, roles, and model backbones. The authors find that skills evolved from diverse multi-model traces substantially outperform single-source skills on cross-model transfer (73.1% vs. 36.0–59.4%), while cross-role transfer can hurt performance due to role-specific over-specialization.

    📌 Why selected: AFTER benchmark: 382 enterprise tasks probing procedural memory reuse across 6 domains

    Training Open Models for Agentic Phone Use¶

    PhoneBuddy is a training recipe for open phone-use agents that combines a real-app RL environment with PhoneWorld, a mock-app environment reconstructed from real GUI traces. A 4B model trained with mixed real+mock RL reaches 45.33% task success on a 150-task real-phone human evaluation (up from 36.67% after SFT) and 83.2% on AndroidWorld, outperforming GPT-5.4 and Seed 2.0 Pro on average. The key finding is that mock-app training complements but does not replace real-app RL.

    When Agents Commit Too Soon: Diagnosing Premature Commitment in LLM Agents¶

    LLM agents running multi-step ReAct loops can silently "lock in" to an interpretation early, a failure mode called premature commitment that final-answer evaluation cannot detect. The paper operationalizes this as cross-run hidden-state convergence (representational commitment), shows it predicts trajectory consistency but not correctness, and demonstrates a runtime monitor (AUROC up to 0.97) plus a prompting intervention that reduces behavioral variance by 28%.

    PlanBench-XL: Evaluating Long-Horizon Planning of LLM Tool-Use Agents in Large-Scale Tool Ecosystems¶

    PlanBench-XL is a benchmark of 327 retail tasks over 1,665 tools that evaluates whether LLM agents can plan across long horizons under partial tool visibility and unreliable tool access. Agents must iteratively retrieve tools, infer implicit sub-goals, and re-plan when critical tool paths are disrupted. Even GPT-5.4 achieves only 51.90% accuracy in default settings, collapsing to 11.36% under the harshest blocking condition.

    Libretto: Giving LLM Agents a Sense of Musical Structure¶

    Libretto is an agent-facing framework that represents symbolic music in an LLM-readable grammar with explicit onset slots and bar-level organization, then evaluates generated pieces against a 29-axis corpus-calibrated structural fingerprint. A bounded generate–measure–revise loop uses that fingerprint to iteratively repair structural failures. This turns symbolic music into a measurable, editable object that LLM agents can inspect and refine without relying on subjective quality scores.

    CalVerT: Augmenting Agents with Calibrated Verifier Telemetry Improves Action and Learning in Knowledge-Intensive Tasks¶

    CalVerT augments LLM agents with two calibrated telemetry signals—a verbal self-confidence score and a per-claim grounding verifier score—inserted into the agent's state at each turn of a ReAct-style loop. This dual-signal approach corrects two symmetric failure modes in knowledge-intensive QA: committing to confidently wrong answers (parametric over-trust) and retrieving redundantly when evidence already suffices (over-retrieval). The method works training-free across existing frameworks and further amplifies gains when used with reinforcement learning.

    📌 Why selected: Stengel-Eskin (UNC); calibrated verifier signals cut over-retrieval and premature stopping

    Counsel: A Meta-Evaluation Dataset for Agentic Tasks¶

    Counsel is the first public meta-evaluation dataset for agentic tasks, providing human meta-annotations of LLM-as-a-Judge (LLMJ) process-level critiques on agent trajectories from τ-bench (customer support) and DA-Code (coding). It enables calibration, training, and benchmarking of LLMJs beyond mere final-judgment accuracy. The strongest judge achieves ~88% agreement with humans on error location and ~65% on reasoning quality.

    📌 Why selected: Meta-evaluation dataset for agentic trajectories; addresses annotation bottleneck at scale

    DataClaw0: Agentic Tailoring Multimodal Data from Raw Streams¶

    DataClaw0 reframes multimodal data curation as a learnable agentic capability: given raw high-entropy streams (long videos, GUI logs, embodied trajectories) and a user intent, a 9B VLM actively filters, reasons over, and restructures the data into dense, schema-compliant supervision. The system is trained via a two-stage pipeline (deterministic anchor extraction → VLM synthesis) and GRPO reinforcement learning with structured rewards. Downstream evaluations show DataClaw0-generated data matches or exceeds Gemini-3.1-Pro-annotated data on GUI navigation, video generation, and spatio-temporal VQA.

    EvoEmbedding: Evolvable Representations for Long-Context Retrieval and Agentic Memory¶

    EvoEmbedding replaces static chunk-level embeddings with a recurrent, context-aware approach: a fixed-capacity FIFO latent memory is updated as segments are processed sequentially, and each embedding is generated by jointly conditioning on the current segment and that evolving memory. A 4B variant outperforms Qwen3-Embedding-8B (+11.1%) and KaLM-Embedding-Gemma3-12B (+6.4%) on long-context retrieval, and a naive RAG pipeline using it surpasses dedicated agentic memory systems (A-MEM, LightMem) with zero extra token overhead.

    OpenRath: Session-Centered Runtime State for Agent Systems¶

    OpenRath proposes Session as a first-class runtime value for multi-agent systems, unifying conversation chunks, tool evidence, sandbox placement, branch lineage, memory events, and token usage into one branchable, inspectable, replayable object. The programming model is structurally analogous to PyTorch — Session flows like a tensor through Agent layers inside Workflow containers — making fork, merge, and replay explicit runtime operations rather than reconstructions from scattered side channels. The paper is a design and architecture report; it explicitly defers broad quantitative evaluation to follow-on work.

    Deep Research in Physical Sciences: A Multi-Agent Framework and Comprehensive Benchmark¶

    PhySciBench is a 200-question expert-curated benchmark covering six physical-science task categories across three cognitive stages, revealing that even the strongest generalist agent (Gemini Deep Research) achieves only 33.5% accuracy. DelveAgent, a modular multi-agent framework built around adaptive planning, dual-granularity memory, and hierarchical physics-grounded reflection, reaches 41.0% on PhySciBench (+7.5 pp) at roughly one-third the inference cost of the strongest baseline.

    DailyReport: An Open-ended Benchmark for Evaluating Search Agents on Daily Search Tasks¶

    DailyReport is a benchmark of 150 open-ended daily search tasks with 3,546 rubrics drawn from real trending topics (Weibo, Reddit, Facebook), designed to evaluate Search Agents (SAs) on everyday user information needs rather than specialist queries. It introduces a user-centric cascade evaluation pipeline that disentangles instruction-following, factuality, and rationality and produces an interpretable user preference score. Across 17 frontier systems, all fall below the "acceptable" threshold (UserPref < 3), with factuality as the dominant failure mode.
