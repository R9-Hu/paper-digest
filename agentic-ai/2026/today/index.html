    🆕 Today's Digest — Agentic AI / LLM Agents¶

    New papers/posts collected 2026-06-17.

    📋 In brief¶

    This batch is dominated by two intertwined pressures: making LLM agents more capable through reinforcement learning, and making them safer as that capability grows. On the capability side, a dense cluster of papers tackles credit assignment and exploration in multi-turn RL (GRPO variants, HGPO, iStar, SPIRAL, LAMER), scalable deep-research agents (ASearcher, EDR, Analytica), and long-horizon computer-use and SWE benchmarks that expose stark gaps between frontier models and human performance. Running in parallel is a maturing safety literature that moves beyond jailbreaks to agent-specific threat surfaces — tool-description injection, decomposition attacks that split harmful goals across benign subtasks, deceptive e-commerce interfaces, and world-model poisoning — with several papers proposing formal monitors, guardrails, and cost-benefit frameworks for keeping agents in check at production scale. Cutting across both tracks is a shared infrastructure concern: standardized data protocols, DAG-structured evaluation pipelines, hierarchical memory systems, and speculative execution are all attempts to tame the engineering complexity of deploying multi-agent systems reliably and efficiently.

    📝 Today's papers¶

    Stealthy World Model Manipulation via Data Poisoning NEURIPS 2026¶

    SWAAP is a two-stage data poisoning framework that attacks learned world models in model-based RL by stealthily manipulating a small fraction of fine-tuning transitions. It first finds an adversarial target world model via bilevel optimization, then realizes it through gradient-matched poisoning that evades standard detectors. The attack causes large performance drops while keeping poisoned data indistinguishable from clean dynamics.

    📌 Why selected: NeurIPS 2026; first systematic poisoning attack on model-based RL world models

    Native Active Perception as Reasoning for Omni-Modal Understanding ICML 2026¶

    OmniAgent is a native omni-modal agent that reformulates long-video understanding as a POMDP-based Observation-Thought-Action (OTA) cycle, selectively distilling audio-visual cues into persistent textual memory so that reasoning complexity scales with query difficulty rather than video duration. Trained via two-stage agentic SFT + a novel RL objective (TAURA), the 7B model achieves state-of-the-art results among open-source models across ten benchmarks, including outperforming Qwen2.5-VL-72B on LVBench (50.5% vs. 47.3%) using 73% fewer frames.

    📌 Why selected: ICML 2026; active-perception reasoning cuts compute for long-video agents

    SWE-Future: Forecast-Conditioned Data Synthesis for Future-Oriented Software Engineering Agents¶

    SWE-Future is a data synthesis pipeline that forecasts likely future repository work directions from pre-snapshot evidence, retrospectively validates those forecasts against later pull requests, and uses validated forecast families as conditioning signals to synthesize coding-agent tasks — bypassing direct historical PR replay. In an 80-repository study, the forecaster achieves 58.1% future-work relevance, and the method produces a 200-task executable coding-agent dataset across 61 repositories.

    📌 Why selected: Addresses benchmark contamination in SWE agents with future-conditioned data synthesis

    What Must Generalist Agents Remember?¶

    This paper proves two necessity theorems characterizing what a generalist agent's memory must encode: (1) whenever two domains require incompatible optimal actions at the same state, a uniformly near-optimal agent's memory must statistically distinguish those domains at that state; (2) if memory is sufficient to predict optimal values for auxiliary one-step probe goals, then a decoder of the local interventional transition kernel exists. These results formally ground memory as the substrate for domain disambiguation and world-model reconstruction.

    📌 Why selected: CMU's Jeff Schneider; formal theory of what generalist agents must store in memory

    STARE: Surprisal-Guided Token-Level Advantage Reweighting for Policy Entropy Stability¶

    STARE identifies that GRPO-style RLVR training suffers from entropy collapse because trajectory-level advantages are applied uniformly to all tokens, systematically over-reinforcing common low-surprisal tokens while under-representing rare high-surprisal tokens that preserve diversity. It proposes a theoretically grounded, minimally invasive fix: selectively reweight the effective advantage of entropy-critical (high-surprisal) tokens, gated by a closed-loop entropy target, to sustain exploration throughout long RL training runs.

    📌 Why selected: Microsoft's Haipeng Luo; principled fix for entropy collapse in GRPO/RLVR training

    RODS: Reward-Driven Online Data Synthesis for Multi-Turn Tool-Use Agents¶

    RODS addresses gradient signal depletion in multi-turn tool-use RL training by dynamically synthesizing new training data targeted at the agent's evolving capability boundary. It repurposes GRPO rollout reward variance—already computed during training—as a zero-cost boundary detector, then generates structurally isomorphic task variants to maintain a saturated learning signal. Starting from 400 human seeds, RODS matches a 17K-sample offline pipeline using ~20× fewer trajectories.

    📌 Why selected: Solves data depletion bottleneck in multi-turn tool-use RL with online synthesis

    Beyond Reward Engineering: A Data Recipe for Long-Context Reinforcement Learning¶

    This paper argues that long-context RL improvements are bottlenecked by training data diversity, not reward design. The authors construct a curated ~14K-example mixture spanning three complementary abilities—retrieval, multi-evidence synthesis, and reasoning—and show that paired with a minimal outcome-based GRPO setup, it consistently outperforms prior long-context RL datasets. Gains transfer to real agentic benchmarks (GAIA, BrowseComp) when applied on top of an agent-tuned model.

    📌 Why selected: Data recipe for long-context RL; directly enables agents reasoning over long trajectories

    Learning User Simulators with Turing Rewards¶

    Turing-RL trains LLM-based user simulators via reinforcement learning using a discriminative Turing-Test reward: an LLM judge scores how indistinguishable a generated response is from the real user's response, rather than measuring content overlap with a ground-truth answer.
