    AGORA: An Archive-Grounded Benchmark for Agentic Workplace Document Reasoning¶

    🕒 Published (v1): 2026-06-23 00:00 UTC · Source: HuggingFace · link

    TL;DR¶

    AGORA is a benchmark for evaluating LLM agents on archive-grounded document reasoning: locating sparse evidence across large, messy workplace document collections (9,664 files, 372M tokens across 8 domains) that exceed any model's context window. The benchmark requires deliberate exploration, cross-file reconciliation of inconsistent terminology/units/conventions, and numeric computation. Even the strongest evaluated model (Gemini-3.1-Pro) reaches only 59.4% accuracy, confirming the task is far from solved.

    Problem¶

    Existing benchmarks fail to jointly stress three properties required for real workplace AI: (1) archive-groundedness (reasoning over a fixed internal corpus, not open web or parametric knowledge), (2) agentic exploration (active file navigation and multi-step tool use because the corpus exceeds any context window), and (3) cross-domain coverage (diverse file formats, terminology, and reporting conventions). Multi-hop QA benchmarks use homogeneous Wikipedia corpora; document QA benchmarks are framed as reading comprehension without navigation; web-agent benchmarks (GAIA, BrowseComp) target open environments rather than bounded archives. The closest prior work, OfficeQA Pro, draws from a single external source, limiting format and domain variety.

    Method¶

    Benchmark construction (three-stage agentic pipeline):

    1. Data collection & preprocessing: Domain seeds are derived from occupational classification systems; a deep-search agent retrieves authentic workplace documents (PDF, Markdown, Excel, CSV) from 8 professional domains. PDFs are converted to Markdown via OCR and chunked by 5-page windows; Markdown files use 8,000-token sliding windows (800-token overlap); Excel/CSV sheets become column-profile summaries. An information-density heuristic selects top-100 seed chunks per domain. GPT generates chunk-, document-, and domain-level summaries encoded into a vector database.

    2. Task synthesis: Given a seed chunk, an agent performs cross-document semantic search, identifies bridging facts, and drafts a candidate multi-hop question with its reference reasoning path and verification code. A refinement stage checks consistency, answerability, and cross-file coverage. An obfuscation stage removes lexical leakage (stem terms that directly retrieve evidence in 1–2 searches) and structural leakage (entities the solver should infer but are stated outright), replacing them with business-role descriptions preserving the original dependency structure.

    3. Quality control: Tasks solvable from parametric knowledge are discarded via closed-book testing. A three-model panel (GPT-5.5, DeepSeek-V4-Flash, DeepSeek-V4-Pro) eliminates tasks all three solve correctly (too easy). An agent checks solvability and unambiguity conditioned on the query alone vs. with the reference reasoning path. Final human annotation yields 362 tasks with unique verifiable numeric answers.

    Evaluation harness: All models run in mini-swe-agent exposing only a bash tool. Tasks execute in isolated E2B sandboxes (no internet, collection mounted locally), capped at 200 turns / 3,600 seconds. Answers are submitted in <answer>...</answer> tags and verified deterministically against gold numeric answers.

    Key Contributions¶

    • AGORA benchmark: 362 questions over 9,664 authentic workplace documents (372M tokens) spanning 8 professional domains (Agriculture, Architecture, Business, Education, Finance, Healthcare, Law, Technology), all exceeding model context windows.
    • Agentic construction pipeline: Cross-document task synthesis with leakage-preventing obfuscation (lexical and structural), difficulty filtering, and human verification—enabling reproducible, deterministic evaluation without LLM-as-judge.
    • Evaluation of 8 frontier models (proprietary and open-weight) under a fixed minimal harness, isolating model-level reasoning from scaffolding artifacts.
    • Cross-domain leaderboard analysis revealing that per-domain rankings diverge from aggregate rankings, exposing systematic model–domain interaction effects invisible in single-source benchmarks.

    Results¶

    • Overall accuracy: Best model (Gemini-3.1-Pro) 59.39%; GPT-5.5 54.70%; GLM-5.1 50.00%; DeepSeek-V4-Pro 45.86%; DeepSeek-V4-Flash 40.06%; Qwen3.5-35B-A3B 11.33%; Gemini-3.1-Flash-Lite 6.35%; Qwen3.5-9B 3.04%.
    • Two-tier split: Five frontier models occupy a 40–60% band; three lower-tier models fall below 12%, with Qwen3.5-9B at or below 3% in five of eight domains—effectively non-functional on AGORA.
    • Per-domain rank inversions: Gemini-3.1-Pro tops 5/8 domains overall but drops to 41.03% on Finance (4th place); GPT-5.5 leads Law and Tech but falls to 38.00% on Business; DeepSeek-V4-Pro overtakes GPT-5.5 on Business despite trailing by 8.8 points overall.
    • Dominant failure modes (frontier models): Incomplete Inspection (II) and Evidence Misidentification (EM) are most prevalent; lower-tier models additionally suffer from Resource Exhaustion (RE) and Instruction Non-Following (INF).
    • No model exceeds 60%, leaving substantial unsolved headroom despite using frontier-tier models at maximum reasoning effort.

    Limitations¶

    • Small total question count (362 questions), with some per-domain sets as small as 29 (Architecture) or 34 (Healthcare), limiting statistical power for fine-grained analysis.
    • Finance domain is heavily skewed by document count (5,966 of 9,664 docs; 117M of 372M tokens), potentially making cross-domain volume comparisons unbalanced.
    • All queries are in English even though source documents may appear in other languages, which may conflate language handling difficulty with reasoning difficulty.
    • Evaluation restricted to numeric answers to enable deterministic verification, excluding open-ended synthesis or free-text deliverables common in real workplace tasks.
    • The minimal bash-only harness (mini-swe-agent) intentionally removes scaffolding to isolate model reasoning, but may underestimate frontier performance achievable with richer agent frameworks.
    • Construction pipeline relies on LLM-generated question drafts; despite obfuscation and human review, residual synthesis artifacts cannot be fully ruled out.

    Relevance to Agentic AI / LLM Agents¶

    AGORA directly benchmarks the core loop of agentic LLM systems—plan, explore, retrieve, reconcile, compute—in a realistic setting where exhaustive context scanning is architecturally impossible, forcing genuine strategic search behavior. The benchmark's finding that per-domain rankings diverge sharply from aggregate rankings is a concrete empirical warning against single-source evaluation, directly relevant to researchers designing agent evaluation suites or selecting models for domain-specific deployment. The deterministic, numeric-answer evaluation design offers a clean methodology for agent benchmarking without LLM-as-judge noise, useful as a template for future agentic benchmarks. The failure-mode taxonomy (Incomplete Inspection, Evidence Misidentification, Resource Exhaustion, Instruction Non-Following, Hallucination) provides a diagnostic vocabulary for understanding where current agents break down in long-horizon document-grounded reasoning.
