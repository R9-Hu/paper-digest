    📋 In brief¶

    This batch is dominated by a maturation-of-infrastructure theme: the agent harness — the runtime scaffold of prompts, memory, tools, and control flow wrapping a model — has become a first-class research object, with papers showing that harness design and model choice each independently shift task performance by comparable margins, and that harnesses can be automatically evolved, compressed, or made learnable. A second major thread is scaling multi-agent coordination, where empirical work finds that degradation as agent catalogs grow is driven by routing confusion and agent-discovery noise rather than task complexity, spurring new architectures for event-driven orchestration, decentralized shared-context execution, and heterogeneous inter-agent communication. Benchmarking quality is itself under the microscope: multiple papers expose that existing benchmarks are gameable (16% of terminal-agent tasks passable without solving them), that oracle-less test patches are the norm in agent-authored code, and that fixed-compute evaluation systematically understates frontier capability — prompting a wave of harder, more realistic evaluation environments spanning long-horizon GUI workflows, enterprise deep research, and CEO-level strategic planning. Cutting across all of this is a security and reliability subplot, with work on defending agentic RAG pipelines against prompt injection, enforcing cryptographic authority boundaries at the API layer, and formally characterizing why detect-and-block defenses collapse under iterative attacker search — signs that the field is beginning to treat deployment safety as a systems problem rather than an afterthought.

    Large Language Models Do Not Always Need Readable Language¶

    This paper investigates whether LLMs can generate and consume compact, non-human-readable textual representations—dubbed BabelTele—that sacrifice readability while preserving semantic content recoverable by instruction-tuned models. BabelTele achieves 99.5% semantic fidelity at 27.9% of original text length via zero-shot prompting, with no fine-tuning required. Experiments show it outperforms natural-language summaries and LLMLingua-2 on accuracy-retention trade-offs and transfers across heterogeneous model families.

    📌 Why selected: Model-to-model compact encoding; could reshape multi-agent communication protocols and token efficiency

    When Does Streaming Tool Use Help? Characterizing Tool-Intent Stabilization in Streaming Retrieval-Augmented Generation¶

    This paper introduces tool-intent stabilization — a query-intrinsic, model-agnostic quantity measuring how early in a streaming input a speculative RAG tool query converges to the answer-bearing retrieval result. On the CRAG benchmark (1,371 questions), sufficiency stabilizes at mean \(\phi_\text{suf} = 0.26\) of the utterance, meaning 73.9% of queries can hide at least 80% of tool latency at a realistic operating point (\(L=600\text{ms}\), \(\delta=3\text{w/s}\), \(\theta=0.8\)). The contribution is measurement and bound-derivation, not a new system.

    📌 Why selected: Characterizes when streaming tool-call speculation helps; concrete latency guidance for RAG agent design

    SoftSkill: Behavioral Compression for Contextual Adaptation¶

    SoftSkill converts natural-language skill documents (Markdown files used to adapt frozen LLM agents) into compact soft prefix embeddings trained via next-token prediction, keeping the backbone frozen and tuning only a soft delta \(\Delta p\). On single-round QA with Qwen3.5–4B, a 32-token prefix surpasses the SkillOpt text-skill baseline by 5.2 points on SearchQA and 12.5 points on LiveMath, while replacing 407–2035 Markdown tokens with 32 virtual tokens. Agentic multi-step tasks show useful but non-robust signal from sparse trajectory imitation.

    📌 Why selected: Behavioral compression of agent skill Markdown files; directly impacts harness skill management

    Navigating Unreliable Parametric and Contextual Knowledge: Explicit Knowledge Conflict Resolution for LLM Inference¶

    MACR is a multi-agent framework for resolving knowledge conflicts in LLM inference, addressing the realistic but understudied scenario where both parametric knowledge and retrieved external context may be simultaneously unreliable. It replaces the dominant "pick one source" paradigm with explicit, interpretable conflict analysis via a three-agent pipeline: Observer, Analyzer, and Reasoner. Empirical results show significant gains over state-of-the-art baselines on knowledge-conflict benchmarks.

    📌 Why selected: Explicit parametric-vs-contextual knowledge conflict resolution; core reliability primitive for RAG agents

    Probe-and-Refine Tuning of Repository Guidance for Coding Agents¶

    Coding agents rely on repository-level guidance files (AGENTS.md/CLAUDE.md), but prior work disagreed on whether such files help or hurt. On SWE-bench Verified with Qwen3.5-35B-A3B, it achieves 33.0% mean resolve rate vs. 28.3% for a static knowledge base and 25.5% for no guidance.

    📌 Why selected: Probe-and-refine tuning of repo-level guidance for coding agents; practical recipe with code

It achieves state-of-the-art attack success rates (ASR) of 81.4% on OpenAI o3 and 67.3% on Claude Opus 4.1 using only black-box API access, improving over prior SOTA by 32%+ with a comparable or smaller query budget.

    StepORLM: A Self-Evolving Framework With Generative Process Supervision For Operations Research Language Models ICLR 2026¶

    StepORLM is a self-evolving training framework for LLMs solving Operations Research (OR) problems, pairing a policy model with a generative process reward model (GenPRM) that co-evolve through a dual-feedback loop. An 8B-parameter model trained under this framework achieves state-of-the-art across six OR benchmarks, outperforming much larger generalist models (DeepSeek-V3 671B, GPT-4o) and specialized baselines. The co-evolved GenPRM also serves as a transferable inference-time verifier that boosts other OR LLMs.

    📌 Why selected: ICLR 2026; StepORLM: self-evolving generative process supervision for OR language models

    ICaRus: Identical Cache Reuse for Efficient Multi-Model Inference ICLR 2026¶

    ICaRus enables multiple task-specialized LLMs to share a single KV cache by decomposing each decoder-only Transformer into a frozen "logical encoder" (KV generator) and a fine-tuned "logical decoder" (token predictor). This eliminates per-model KV cache duplication in multi-model agentic workflows, achieving up to 11.1× lower P95 latency and 3.8× higher throughput compared to conventional multi-model serving.

    📌 Why selected: ICLR 2026; ICaRus: identical KV-cache reuse across multi-model agentic inference
