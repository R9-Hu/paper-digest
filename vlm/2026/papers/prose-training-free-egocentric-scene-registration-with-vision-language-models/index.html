    PROSE: Training-Free Egocentric Scene Registration with Vision-Language Models¶

    🕒 Published (v1): 2026-06-15 11:11 UTC · Source: Arxiv · link

    TL;DR¶

    PROSE is a training-free, RGB-only pipeline that registers two egocentric captures of the same indoor scene across time by using a pretrained VLM both to build per-scan 3D scene graphs and to match object instances across scans. Height-binned Set-of-Marks prompting with a paired same/different verification pass produces reliable cross-scan correspondences without any fine-tuning. On Aria Digital Twin and Aria Everyday Activities benchmarks, it outperforms both geometric and learned scene-graph baselines on registration recall and error.

    Problem¶

    Cross-time egocentric scene registration from RGB-only head-mounted cameras is unsolved at scale: classical geometric methods require clean, dense point clouds that monocular egocentric video cannot reliably produce, and learned scene-graph methods (SGAligner, SG-PGM, SG-Reg) assume curated 3D inputs or pre-built graphs and fail to transfer to the egocentric distribution. No prior method jointly produces the scene graph and the cross-scan correspondences from raw RGB.

    Method¶

    Scene parsing. VGGT-Ω predicts per-frame depth, intrinsics, and camera pose from RGB; frames are back-projected into a per-subscan point cloud with a 2D-to-3D pixel index. A pretrained VLM (Qwen3.6-27B) is prompted with subsampled frames and returns up to 20 object names using five stacked prompt modifiers: landmark-salience, static-rigid focus, hallucination guard, occlusion recovery, and scene prior. SAM 3 performs open-vocabulary video segmentation from those names, yielding temporally consistent per-frame instance masks. A voxel-majority-vote fusion step resolves cross-frame ID inconsistencies and merges near-duplicate instances into a scene graph G = (V, E).

    Cross-scan correspondence. Instances from G_ref and G_src are height-binned into K=5 equal-count quantile bands with 20% overlap. Within each bin, the VLM receives two panels of object crops labeled with a shared-namespace Set-of-Marks grid (reference markers 1…N, source markers N+1…N+M) and proposes matched pairs. Each candidate is then verified by two prompts — "are these the same?" and "are these different?" — keeping only pairs where both answers are consistent.

    Pose estimation. For each matched object pair, RANSAC is run on per-instance point-level descriptors (FCGF, FPFH, or GeoTransformer, all off-the-shelf) to produce a candidate rigid transform T_i ∈ SE(3). The candidate with the highest inlier ratio against the full source cloud is selected as the final transform.

    Key Contributions¶

    • Training-free, RGB-only cross-time egocentric scene registration that jointly recovers instance correspondences, an open-vocabulary scene graph, and a rigid SE(3) transform.
    • Height-binned Set-of-Marks VLM prompting as a cross-scan correspondence engine — no learned matcher required.
    • Paired same/different verification pass that eliminates hallucinated VLM matches with sharp precision gain at small recall cost.
    • Per-instance RANSAC hypothesis generation with inlier-ratio voting, making pose estimation robust to individual mismatched pairs.
    • Demonstration of scene-graph transferability to downstream path planning on a Spot robot.

    Results¶

    Scene registration (ADT, GT point clouds): - PROSE (FCGF): RR 89.2%, RRE 4.16°, RTE 0.13 m vs. best scene-level baseline TEASER++/FPFH: RR 76.2%, RRE 12.31°, RTE 0.54 m; vs. SG-Reg: RR 47.8%, RRE 20.74°, RTE 1.97 m. - PROSE (GeoTrans): RR 81.2%, still outperforming all baselines on rotation and translation error.

    Scene registration (ADT, RGB-reconstructed clouds): - PROSE (FCGF): RR 50.6%, RRE 11.87°; SG-Reg collapses to RR 20.7%; scene-level methods degrade proportionally more than PROSE, confirming semantic correspondence is more noise-tolerant.

    Scene-graph correspondence (ADT): - PROSE: NP 70.4%, NR 49.6%, F1 56.9%, CR 98.6% vs. GDino+CLIP: F1 17.2%, CR 89.7%; SG-Reg: F1 25.0%, CR 76.9%.

    Ablation (correspondence, Tab. 3b): - Full pipeline F1 0.45, CR 1.0; removing height bins drops F1 to 0.29 and CR to 0.58 (largest single contributor); removing double verification drops F1 to 0.36.

    Object listing (Tab. 3a): - Stacking all five prompt modifiers raises anchor recall from 71.2% (base) to 92.4% with final precision 95.8%.

    Path planning: - PROSE yields 192 valid nodes and 244 in-room pairs; ConceptGraphs yields 33 nodes / 234 pairs; FM-Fusion yields 5 nodes / 2 valid pairs. PROSE achieves 49.6% in-room success rate vs. ConceptGraphs 46.2%.

    Limitations¶

    • VLM spatial reasoning is sensitive to prompt phrasing; robustness expected to improve only as 3D-pretrained VLMs mature.
    • Relies on static landmark objects; degrades under rapid camera motion or sparse scenes with few distinctive objects.
    • Multi-stage pipeline means errors compound: failures in geometry, segmentation, or correspondence each propagate downstream.
    • AEA evaluation uses only VGGT-Ω-predicted clouds (no GT clouds available), making direct comparison with GT-cloud results on ADT indirect.
    • Runtime not reported in the main paper; inference on a single H200 GPU with RANSAC (10K iterations) per matched pair may be slow for large scene pairs.

    Relevance to Vision-Language Models¶

    PROSE demonstrates a concrete, quantitatively validated use case for VLMs as zero-shot spatial reasoners: the same pretrained model performs open-vocabulary scene listing and cross-instance disambiguation without any domain-specific fine-tuning. The height-binned Set-of-Marks prompting strategy is a transferable technique for constraining VLM attention to tractable, geometrically filtered subsets — directly applicable to other 3D grounding and multi-view correspondence tasks. The paired same/different verification scheme is a lightweight but effective hallucination mitigation pattern that extends beyond registration. For researchers tracking VLMs, this work bridges the gap between single-scene 3D VLM grounding (ConceptGraphs, HOV-SG, GPT4Scene) and the harder cross-scan setting, establishing a new capability frontier for embodied scene understanding.
